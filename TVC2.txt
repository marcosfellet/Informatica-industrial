- Sistemas Distribuídos
    - Modelo OSI
        - Camada física
            - Bits para sinais físicos
        - Camada de Enlace
            - Endereçamento físico do dispositivo
            - Códigos de checagem e controle de erros
            - Montagem do frame de transmissão (pacote pra envio)
            - CRC
                - Calculo de todos os bits transmitidos => número
                - Calculo inverso
                - Numeros diferentes => bits invertidos
        - Camada de rede
            - Intercomunicação de dispositivos e nós em uma rede lógica
            - Roteamento e endereçamento lógico
            - Dispositivos não precisam estar no mesmo ambiente
            - Protocolo IP

        - Camada de Transporte
            - Conexão entre dispositivos
            - Protocolos TCP e UDP
                - TCP - confiabilidade (maior gasto de banda)
                - UDP - velocidade
        - Camada de Aplicação
            - reconhecimento facial, calculadora, MODBUS, etc

    - Arquitetura de sistemas distribuídos
        - Cliente-servidor
            - Servidor é um elemento passivo da rede 
              (não atua se não for provocado)
    - API Socket
        - primitivas
            - Bind
                - Informar o sis.op que deseja utilizar o IP e uma porta 
            - Listen
                - Recebe as conexões
                - Cliente tenta se conectar antes do listen, sis.op nega o acesso 
            - Acess  
                - fornece o endpoint para a conexão 

- Programação Concorrente
    - Tarefas segmentadas para fazer todas as tarefas 
    - OI-bound
        - Pouco tempo de processamento e muito tempo de espera
    - CPU-bound 
        - Muito tempo de processamento
        - Jogo, simulação, etc 
    - Thread
        - Linha de execução 
        - Fluxo único de execução 
        - Uma instrução vem depois da outra
        - Multithread
            - Desacoplamento das execuções
            - Deve ser usado:
                - problemas IO-bound
            - Não deve ser usado:
                - CPU-bound 
            - Condição de corrida
                - Variável compartilhada por 2 ou mais threads
                    - Geralmente são atributos de classe compartilhados
                    - Lock
                    - Semáforo 
                        - limitar recurso
                        - caso genérico do lock
                        - mais de uma thread pode ser executada
